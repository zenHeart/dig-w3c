**前言：讲解在 js 中使用正则的注意事项及和标准正则语法相比存在的区别和因该注意的问题**
## js 正则对象的创建
js 中正则分为两部分如下   
`<pattern><flags>`  
1. pattern 是正则语法部分，js 支持的正则语法后面有详细讲解
2. flags 这一部分表示正则语法的搜索模式的配置项，参数如下 
  
**正则支持的配置选项**     

标志位 | 含义
:---|:---
g| 表示全局匹配
i| 表示匹配是不区分大小写
m| 多行匹配模式，对于`^` 和 `$` 以换行或回车符表示起始和结束，而不是整个文本的开始和结束
y| 表示从指定的位置开始匹配字符串，而不进行循环匹配
u| 使用 Unicode 模式进行匹配


支持如下两种模式的正则定义方法：
1. 字面量模式定义
```js
   //斜杠内为正则模式，后面为正则配置项
   var  reg = /test/g; //斜杠内为正则的匹配模式,后面为配置模式
```
2. 使用构造函数创建
```js
   // 第一个参数为正则模式后面为正则配置项
   vat reg = new Regexp("test","g") // 引号内为正则规则
```
<span style="color:red">注意在使用构造函数表示模式时，若表示模式时需要经过转义字符，
而第一种定义方式不需要</span>
```js
   var reg = /\w/g ; //表示全局搜索 字符字母或下划线 
   
   //错误的写法
   var regError = new RegExp('/\w/','g') ; //该模式只是搜索 w 字母
    
  //正确写法
  var regRight = new RegExp('/\\w/','g'); // 注意这里需要经过两次转义才对
```

### js 支持的正则表达式
类别|字符 | 含义
:---|:---|:---
转义|\| 转义字符，用来实现对正则字符或其他非文本字符的索引
字符类别|`[xyz]`| 匹配符合括号类字符集合的单个字符，注意其他特殊的字符在该模式下会被当做普通字符进行处理
字符类别|`[^xyz]`| 于上面的含义相反，匹配不属于该字符集的单个字符
字符类别|`n｜m`|逻辑或符号实现表示匹配模式 n 或 m 均可  
字符类别|`[\b]`| 只匹配(u+0008) 暂时不知道作用
字符类别|.|匹配单个字符，除换行符以外
字符类别|\cX| 匹配 ctrl + X 例如、\cm 或 \cM 匹配 CTRL + `[mM]` 
字符类别|\d| 匹配数字，相当于 `[0-9]`
字符类别|\D| 选择非数字，相当于 `[^0-9]`
字符类别|\f| 匹配换页符号
字符类别|\n| 1. 匹配换行符 2.在匹配组中用于索引组别，后面有介绍
字符类别|\r| 匹配回车符
字符类别|\s| 匹配空白字符，包括回车、换行、空格、制表、换页、换行
字符类别|\S| 非空白字符
字符类别|\t| 匹配制表符
字符类别|\v| 匹配垂直制表符
字符类别|\w| 匹配字母数字下划线类似于 `[a-zA-Z0-9_]`
字符类别|\W| 匹配非字符类似于 `[^a-zA-Z0-9]`
字符类别|\0| 匹配空字符(u+0000) 注意后面跟数字表示匹配 8 进制注意序列
字符类别|\xhh| 匹配 asicc 编码字符
字符类别|\uhhhh| 匹配 utf-16 编码字符
匹配次数|{n,m}|匹配字符或模式 n 到 m 次,注意 m 不填时表示大于 n 即可，但是 n 必须存在
匹配次数|{n}|匹配字符或模式 {n,n}
匹配次数|?|1. 匹配字符或模式{0,1} 　2.　用在其他次数匹配后表示尽量少的满足匹配条件　3. 结合括号形成预查功能
匹配次数|*| 匹配字符或模式 {0，n}
匹配次数|+| 匹配字符或模式 {1，n}
匹配组|(x)| 匹配组，用圆括号围成的模式会被保存成数组，按照从左到右的顺序，利用 \1、\2...的顺序进行索引
匹配组|\n| 用来索引匹配组，比如 (w)\1 表示搜索符合 ww 格式的字符，常用来进行重复模式的索引 ([ab])\1 搜索 aa,或 bb 
匹配组|(?:x)|于上面的意义相反，表示不保存某个模式
条件查询|x(?=y)|正向肯定预查，寻找符合 x 后面是 y 模式的字符匹配
条件查询|x(?!y)|正向否定预查，寻找符合 x 后面不是 y 模式的字符匹配
位置匹配|\b|匹配单词的边界，注意这里的单词是指字母数字和下划线组成的字符串，若要匹配英语单词采用`\b[a-zA-Z]+\b`实现
位置匹配|\B|匹配非单词边界,及所有字母数字下划线后面的位置都算非单词边界
位置匹配|^|用来匹配文本开头位置，在开启多行匹配模式后，用来匹配每行的开头
位置匹配|$|用来匹配文本末尾位置，同上开启多行匹配后，可以匹配每行的行末

### 重点正则符号的讲解
**匹配组 `()`**
当使用括号来搜寻正则时，搜寻到的结果会被正则引擎保存，并可以利用反斜杠加数字的方式进行索引。
对于替换时，可利用 `$1`、`$2`...来表示匹配数组，作用和反斜杠模式相同。

```
   (w)\1 //由于字符 w 被圆括号括起来，所以可以利用 \1 来引用括号内的结果，该模式表示匹配两个 w 的字符，类似于 w{2} 
   (w(w))\1\2 //首先匹配组是按照，从左至右的顺序确定的，所以，最外层括号对应 \1 内层对应 \2.
              //由于 \1 匹配外层表示 w{2} \1 表示 w{1},则该匹配等价于 w{5}
   注意若不想保存某个匹配分组，采用 (?:) 的形式，比如
   (w(?:w))\1\2 //此时 \2 的匹配分组是不存在的
   
```
**字符集 `[]`** 
用来匹配字符集，在方括号当中使用圆括号，不会产生对应正向预查、匹配组等功能。而是会当做单独字符处理。
所有特殊字符在字符集中都当做对应字符处理没有特殊意义。
可以利用`-`来表示一个字符范围。注意该模式匹配的是一个字符，若要匹配多个字符需要结合次数符号。
注意由于是字符集需要知道匹配的字符集情况，js 下是 UTF-8 的字符集，别的语言可能有区别。
```
   `[w(?=w)]`   //由于字符集只匹配字符，所以改正则并没有正向预查功能，只表示匹配 w、(、?、=、）这几个字符的意思
   `[w*2]`      //道理同上，相当于匹配 w、*、2 
   [ab]{2}    //由于字符集只能匹配单一字符，所以要匹配两个字符，后面要加数量匹配符，该正则表示匹配 aa、ab、bb 均可，
```

**位置匹配 \b和 \B**
在正则中，位置匹配不同于模式，它是用来实现对位置的查找的。利用 PHPSTORM 来说明位置的含义。
![](../img/js_regexp1.png)   

通过对不同符号类型进行的测试可知，`\b` 对应的位置相当于非文字字符和数字、下划线组成的字符串前后位置。
特殊字符的前后并不属于 `\b` 的位置(在 js 引擎的处理结果可能不同)。

对于`\B`结果如下  
![](../img/js_regexp2.png)   
可以看到，`\B` 的结果和小 b 恰恰相反

**条件匹配 (?=) 和 (?!)**
这两种模式使用为正向肯定和否定预查
```js
    // 该函数的作用是用来实现对后面跟有function的test进行查询
    //比如 testFunction 中的test,注意匹配的是 test 而没有后面的部分
    //js 只支持正向的肯定和否定预查，实际上正则语法中还有反向的肯定和否定预查
    var findTest = /test(?=Function)/g; 
```

**匹配组 `\n`**

`\n` 代表按照匹配组顺序索引到的结果。

例如 

```js
//该模式会查找到 af:af 的字符串
/(([0-9a-f]{2}):)\1/g.exec('af:af:23:45'); 
```

在使用过程中可能将这种指代认为是代表索引模式。
及`\2` 表示 `([0-9a-f]{2})` 的匹配组。
**注意 \2 只代表匹配后的结果**。


## js 正则 DEVICE_API 详解
**属性**

方法|描述
:---|:---|
RegExp.prototype.global|对应正则配置项中的 g,只读
RegExp.prototype.ignoreCase|对应 i,
R.prototype.lastIndex|改值表示起始的搜寻位置可写，注意改值在某些情况下的差异[lastindex](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/RegExp/lastIndex)
RegExp.prototype.multiline|对应 m
RegExp.prototype.source|对应 u
Regexp.prototype.sticky|对应粘滞模式，什么鬼不懂
//important[locke]  粘滞模式不是太懂

**方法**
方法|描述
:---|:---|
exec|利用正则在字符串中查找匹配数组
test|测试字符串中是否有符合条件的正则模式
toString|返回该正则的字面量模式定义

### 重点方法讲解
**test**
该函数用来测试字符串是否符合正则要求，常用此函数进行各种验证。
返回的结果为 bool 值。
```js
   //测试字符串中是否含有，注意这里的匹配模式为全局匹配
   //若去掉全局匹配则只要字符串中的子字符串含有规定模式也会返回真
   //所以在实际使用时注意区分，是需要局部匹配还是全局匹配
   var testPhoneNumber = /\d{11}/g; 
   testPhoneNumber.test('123223233333');
```

### 字符串中正则 DEVICE_API 的使用
**exec**
该函数用来查找符合正则模式的字符串，但是只返回首次匹配的结果举例如下
```js
    // Match "quick brown" followed by "jumps", ignoring characters in between
    // Remember "brown" and "jumps"
    // Ignore case
    var re = /quick\s(brown).+?(jumps)/ig;
    var result = re.exec('The Quick Brown Fox Jumps Over The Lazy Dog');
```
返回值为一个关联数组
* result[0] 表示匹配的整个结果  [0]->`	Quick Brown Fox Jumps`
* result[1],...[n] 表示从左到右匹配组匹配的结果 [1]->Brown [2]->Jumps
* result.index 表示首次匹配到的字符串基于原始字符的索引位置 .index->4
* result.input 表示原始字符， .input = 'The Quick Brown Fox Jumps Over The Lazy Dog'

经过上面的函数后，正则对象`re`的属性发生了如下变化
* re.lastIndex 下一次匹配的开始位置, 25
* re.ignoreCase	是否使用了'i'标记使正则匹配忽略大小写, true
* re.lobal	是否使用了'g'标记来进行全局的匹配, true
* re.multiline	是否使用了'm'标记使正则工作在多行模式（也就是，^ 和 $ 可以匹配字符串中每一行的开始和结束（行是由 \n 或 \r 分割的），而不只是整个输入字符串的最开始和最末尾处。）, false
* re.source	正则模式的字符串	quick\s(brown).+?(jumps)

**使用举例**
1. 查找字符串中所有匹配字符
```js
   var myRe = /ab*/g;
   var str = 'abbcdefabh';
   var myArray;
   while ((myArray = myRe.exec(str)) !== null) {
     var errMsg = 'Found ' + myArray[0] + '. ';
     errMsg += 'Next match starts at ' + myRe.lastIndex;
     console.log(errMsg);
   } 
```
